  User gõ:  /bmad-brainstorming

  - Output: tài liệu brainstorming trong _bmad-output/planning-artifacts/
  - Dùng khi: chưa rõ ý tưởng, cần khám phá

  Bước 1b — Research (tuỳ chọn, chạy song song được)

  User gõ:  /bmad-domain-research
  User gõ:  /bmad-market-research

  - Mỗi lệnh tạo 1 báo cáo nghiên cứu chuyên sâu
  - Output: research documents trong planning-artifacts/ hoặc docs/
  - Dùng khi: cần data trước khi viết brief/PRFAQ

  Bước 1c — Product Brief HOẶC PRFAQ (bắt buộc 1 trong 2)

  Đường B:  /bmad-prfaq

  Bước 2a — Tạo PRD

  User gõ:  /bmad-create-prd
  - Agent: John (PM) — hỏi "WHY?" liên tục
  - Workflow gồm 12 steps tuần tự: init → discovery → vision → executive summary → journeys → domain →
  innovation → project type → scoping → functional → nonfunctional → polish → complete
  - Mỗi step có menu, user chọn Continue để sang step tiếp
  - Output: planning-artifacts/prd.md
  - Đây là document bắt buộc, dài nhất trong pipeline

  Bước 2b — Validate PRD

  User gõ:  /bmad-validate-prd
  - 13 validation steps: format detection → parity check → density → brief coverage → measurability →
  traceability → implementation leakage → domain compliance → project type → SMART → holistic quality →
  completeness → report
  - Output: validation report với danh sách issues

  Bước 2c — Edit PRD (nếu cần)

  User gõ:  /bmad-edit-prd
  - Đọc validation report, sửa PRD theo findings
  - Output: updated PRD

  Bước 2d — UX Design (tuỳ chọn, khuyến nghị cao nếu có UI)

  User gõ:  /bmad-create-ux-design
  - Agent: Sally (UX Designer)
  - 14 steps: init → discovery → core experience → emotional response → inspiration → design system →
  defining experience → visual foundation → design directions → user journeys → component strategy → UX
  patterns → responsive/accessibility → complete
  - Output: planning-artifacts/ux-design.md

  Bước 3a — Architecture

  User gõ:  /bmad-create-architecture
  - Agent: Winston (Architect)
  - 8 steps: init → context → starter → decisions → patterns → structure → validation → complete
  - Output: planning-artifacts/architecture.md
  - Bao gồm: tech stack, API design, database schema, deployment, ADRs

  Bước 3b — Epics & Stories

  User gõ:  /bmad-create-epics-and-stories
  - 4 steps: validate prerequisites → design epics → create stories → final validation
  - Đọc PRD + Architecture + UX → tách thành epics, mỗi epic có stories với acceptance criteria (BDD
  format)
  - Output: planning-artifacts/epics.md
  - Đây là cầu nối giữa planning và implementation

  Bước 3c — Check Readiness

  User gõ:  /bmad-check-implementation-readiness
  - 6 steps: document discovery → PRD analysis → epic coverage → UX alignment → epic quality → final
  assessment
  - Kiểm tra: PRD ↔ Architecture ↔ UX ↔ Epics có aligned không
  - Output: readiness report — GO / NO-GO

  Bước 3d — Project Context (cho brownfield)

  User gõ:  /bmad-generate-project-context
  - Scan codebase hiện tại, tạo project-context.md cho AI agents tham khảo
  - Output: _bmad-output/project-context.md

  Bước 4a — Sprint Planning

  User gõ:  /bmad-sprint-planning
  - Đọc tất cả epic files, parse stories
  - Tạo implementation-artifacts/sprint-status.yaml:
  development_status:
    epic-1: backlog
    1-1-user-authentication: backlog
    1-2-account-management: backlog
    epic-1-retrospective: optional
    epic-2: backlog
    2-1-dashboard: backlog
    ...
  - Chạy 1 lần, sau đó các agent tự cập nhật file này

  Bước 4b — Story Cycle (lặp)

  Tạo story:
  User gõ:  /bmad-create-story
  - Tự tìm story backlog đầu tiên trong sprint-status.yaml
  - Hoặc chỉ định: /bmad-create-story 2-3 (epic 2, story 3)
  - Đọc epics + PRD + architecture + UX + previous story learnings + git history
  - Tạo file implementation-artifacts/1-1-user-authentication.md với:
    - User story + acceptance criteria
    - Tasks/subtasks (checkboxes)
    - Dev Notes (architecture guardrails, library versions, file structure)
    - Previous story intelligence
  - Cập nhật sprint-status: backlog → ready-for-dev

  Validate story (tuỳ chọn):
  User gõ:  /bmad-create-story validate
  - Kiểm tra story file đã đủ context chưa

  Implement story:
  User gõ:  /bmad-dev-story
  - Hoặc: /bmad-dev-story [path-to-story-file]
  - Agent: Amelia (Developer)
  - Tự tìm story ready-for-dev đầu tiên
  - Thực thi red-green-refactor cho từng task:
    a. Viết test FAIL (red)
    b. Code tối thiểu để test PASS (green)
    c. Refactor
    d. Check checkbox [x] trong story file
    e. Lặp cho task tiếp theo
  - Cập nhật sprint-status: ready-for-dev → in-progress
  - Khi tất cả tasks xong: in-progress → review
  - Chạy liên tục không dừng cho đến khi hoàn thành hoặc HALT

  Review code:
  User gõ:  /bmad-code-review
  - Spawn 3 reviewers song song:
    a. Blind Hunter — chỉ thấy diff, không context → tìm bugs thuần túy
    b. Edge Case Hunter — diff + project access → tìm edge cases
    c. Acceptance Auditor — diff + spec + context → kiểm tra acceptance criteria
  - Triage findings thành: MUST FIX / SHOULD FIX / CONSIDER / DISMISSED
  - Nếu Approve → user cập nhật status thành done
  - Nếu Changes Requested → ghi vào story file, user chạy lại /bmad-dev-story

  Bước 4c — Retrospective (cuối mỗi epic)

  User gõ:  /bmad-retrospective
  - Review epic đã hoàn thành: what went well, what didn't, lessons learned
  - Output: retrospective document
